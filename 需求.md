一、核心功能模块
1. 图片上传与预览
上传方式：
本地上传：点击 “选择本地图片” 按钮，支持 JPG/PNG（单张≤5MB），上传后自动压缩（保持比例，压缩至 5MB 内）。
URL 上传：输入框支持 HTTP/HTTPS 图片链接，点击 “确认 URL” 后自动校验（无效链接提示 “请输入有效的图片 URL”）。
预览功能：
上传成功后，在上传区域下方显示预览图（固定尺寸，如 300x200px，超出部分裁剪，不支持缩放 / 下载）。
预览图下方显示 “预览图” 文字标识（避免用户混淆）。
交互限制：两种上传方式互斥（选择本地则 URL 框 + 按钮禁用，输入 URL 则本地按钮禁用），上传成功后允许 “重新上传”（覆盖当前图片）。
2. promptType 选择框
位置：预览图下方，单选下拉框。
选项：按顺序显示 “Midjourney”“Stable Diffusion”“Flux”“通用基础版”，对应值为midjourney/stableDiffusion/flux/normal。
校验：必填项，未选择时 “生成提示词” 按钮置灰，鼠标悬停提示 “请选择提示词类型”。
3. 调用 Coze 工作流 API 生成提示词
触发条件：用户完成图片上传（且预览成功）+ 选择promptType后，点击 “生成提示词” 按钮。
API 请求参数：严格按照你提供的结构组装：
json
{
  "userQuery": "请描述一下这个图片", // 固定值，不允许用户修改
  "img": 图片二进制数据, // 本地图片直接取二进制；URL图片需先下载为二进制（后端处理）
  "promptType": 用户选择的类型值 // 如"midjourney"
}
状态提示：
调用中：显示 “生成中...” 文字（无动画）。
成功：隐藏 “生成中”，显示结果文本框。
失败：显示 “生成失败，请重试”（不区分具体错误原因，如 API 超时、图片无效等）。
4. 结果展示与复制
结果文本框：只读，显示 Coze API 返回的提示词（直接展示原始结果，不做二次处理）。
复制功能：文本框右侧 “复制” 按钮，点击后提示 “已复制”（3 秒后自动消失）。

5. Coze 工作流 API 调用文档（前端版）

1. 接口说明
功能：通过上传图片或图片 URL，获取对应模型（Midjourney/Stable Diffusion 等）的提示词
请求方式：POST
接口地址：https://api.coze.com/v1/workflows/run（请以实际提供的地址为准）
Content-Type：multipart/form-data（因包含图片二进制数据）
2. 前置条件
已获取个人访问令牌（Personal Access Token），格式为 Bearer {token}
前端需支持 FormData 格式（用于封装图片和参数）
推荐使用 HTTP 客户端：trae、axios 或原生 fetch
3. 请求参数
参数名	类型	说明	是否必填
userQuery	String	固定值："请描述一下这个图片"（用于引导工作流解析图片）	是
img	File / Blob	图片二进制数据（本地上传的图片文件，或通过 URL 下载的 Blob 对象）	是
promptType	String	提示词类型，可选值：
"midjourney"、"stableDiffusion"、"flux"、"normal"	是
